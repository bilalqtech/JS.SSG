These are the commands that you can use with JS.SSG.

## Basic command

```
yarn jsssg
```

Builds a production-ready static site in the [out](/configuration#out) directory (`/build` by default)

## Optional arguments

### `--serve`

Running `yarn jsssg --serve` builds a production-ready static site in the [out](/configuration#out) directory, and serves it locally for preview at [http://localhost:8080/](http://localhost:8080/)

### `--port`

Sets the port that JS.SSG uses to serve the preview of the site.

Running `yarn jsssg --serve --port=1337` builds a production-ready static site in the [out](/configuration#out) directory, and serves it locally for preview at [http://localhost:1337/](http://localhost:1337/)

### `--watch`

Runs a build of the site, and then watches the "content" folder for changes. Rebuilds the site when any content files are changed.

### `--verbose`

Adding `--verbose` to the command will include verbose logging in the console.

```bash
$ jsssg
Generating static site...
Site generated at "/build"
✨  Done in 2.63s.
```
